# Command: split-tasks

**角色**：Tech Lead Agent
**用途**：基于接口规范和 ADR，将一个 Sprint 的功能需求拆分为具体的后端/前端开发任务列表，每条任务有明确的描述、依赖关系和验收条件。

**调用方式**：
```
/split-tasks [--be | --fe | --all] [sprint-id]
```

**参数说明**：
- `--be`：仅输出后端任务
- `--fe`：仅输出前端任务
- `--all`（默认）：输出后端和前端全部任务
- `sprint-id`：Sprint 编号，例如 `S03`；如未提供，读取 `workflow/03-sprint-plan.md` 中的当前 Sprint

---

## Step 1：读取输入文档

按以下顺序读取文件，全部读取完毕后再进行拆分，不允许只读部分文件就开始输出。

### 必读文件

1. **Sprint 计划**
   路径：`workflow/03-sprint-plan.md`（或由参数指定的 sprint 文件）
   目的：明确本 Sprint 的交付范围、优先级和排期约束。

2. **接口规范**
   路径：`docs/api/`（读取所有与本 Sprint 功能相关的接口文档）
   目的：从接口定义推导后端任务（实现接口）和前端任务（调用接口、处理响应）。

3. **ADR 文档**
   路径：`docs/adr/`（读取所有状态为 `Accepted` 的 ADR）
   目的：确保任务描述中引用正确的技术决策，避免任务与已决策的架构冲突。

4. **需求文档**
   路径：`docs/requirements/`（读取与本 Sprint 对应的需求文件）
   目的：理解接口背后的业务语义，补充接口规范中未体现的业务规则。

### 可选文件（如存在则读取）

- `docs/gap-registry.md`：检查是否有与本 Sprint 相关的未关闭 Gap，需在任务中体现修复工作。
- `docs/data-model/`：了解数据模型，辅助判断后端任务的数据库操作范围。

---

## Step 2：拆分逻辑

### 2.1 从接口推导后端任务

对每一个接口端点（endpoint），生成对应的后端任务。推导规则：

1. **一个端点至少对应一条后端任务**，包含：
   - Controller 层实现（路由定义、请求/响应映射）
   - Service 层实现（业务逻辑）
   - Repository/DAO 层实现（数据读写，如有）

2. 如果多个端点共享同一个 Service 方法或数据表操作，可合并为一条任务，但必须在任务描述中列出所有涉及的端点。

3. **识别横切任务**：认证鉴权、日志、异常处理、数据校验等如果是公共的，单独作为一条任务，不重复出现在每个端点任务中。

4. **识别基础设施任务**：数据库 Migration、缓存配置、外部服务集成等，单独列为任务。

### 2.2 从接口推导前端任务

对每一个接口端点，生成对应的前端任务。推导规则：

1. **页面级任务**：如果多个接口共同支撑一个页面或组件，合并为一条前端任务，列出调用的所有接口。

2. **状态管理任务**：如果前端需要维护复杂状态（如分页、筛选、实时更新），单独列为一条任务。

3. **错误处理任务**：接口错误码的前端呈现逻辑（Toast、错误页、重试）如果是公共的，单独列为一条任务。

### 2.3 识别依赖关系

按以下规则识别任务间的依赖：

- **数据依赖**：任务 B 的输入需要任务 A 的输出（如前端任务依赖后端接口实现完成）。
- **顺序依赖**：任务 B 只有在任务 A 完成后才能开始（如 Migration 必须在 Service 实现之前）。
- **并行可能**：没有依赖关系的任务标记为可并行执行。

每条任务的 `depends-on` 字段用任务 ID 列表表示，空列表表示可立即开始。

### 2.4 估算工作量

工作量以"人天"为单位，按以下标准估算：

| 规模 | 估算 | 典型场景 |
|------|------|----------|
| XS | 0.5 天 | 简单 CRUD 端点、配置变更 |
| S | 1 天 | 含业务逻辑的单接口实现 |
| M | 2 天 | 多接口协作、复杂查询、状态机 |
| L | 3 天 | 跨模块集成、外部服务对接 |
| XL | 5 天 | 需要拆分，标记并提示人工复核 |

如果某条任务估算为 XL，必须在任务描述后附加警告：`[WARNING] 此任务估算超过 3 天，建议进一步拆分后再确认。`

## 命令行为说明

- 如果指定 `--be`，则只输出"后端任务"部分，跳过"前端任务"部分。
- 如果指定 `--fe`，则只输出"前端任务"部分，跳过"后端任务"部分。
- 如果指定 `--all` 或未指定参数，输出全部任务。
- 任务 ID 统一编号：后端任务以 `BE-` 前缀，前端任务以 `FE-` 前缀，三位数字序号（`BE-001`、`FE-001`）。
- 依赖关系中跨类型任务用完整 ID 引用（如前端任务依赖 `BE-001`）。
- 如果读取文件过程中发现文件不存在，停止执行并输出错误信息，列出缺失的文件路径，不允许跳过缺失文件继续拆分。
