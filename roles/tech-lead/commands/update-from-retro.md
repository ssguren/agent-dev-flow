# Command: update-from-retro

**角色**：Tech Lead Agent
**用途**：将复盘结论写入 Gap Registry（新增预防性条目）和项目 CLAUDE.md（更新规范）。在执行任何写入前，必须向人工展示完整变更摘要并等待确认。

**调用方式**：
```
/update-from-retro <retro-file-path>
```

**参数说明**：
- `retro-file-path`：复盘文档路径，例如 `docs/retro/retro-sprint-3.md`。必须提供，不可省略。路径不存在则报错退出，不猜测默认路径。

**铁律**：Agent 不允许过滤掉任何措施。第 8 节中的每一条预防措施都必须完整展示给人判断，不允许以"不重要"或"已有类似条目"为由静默跳过任何一条。

---

## Step 1：读取复盘文档，识别预防措施

### 1.1 读取文件

按以下顺序读取，全部读取完毕后再开始分析：

1. **复盘文档**（命令参数指定的路径）
   完整读取，不跳过任何章节。如果文件不存在，立即报错退出：
   ```
   错误：复盘文档不存在：<retro-file-path>
   请确认路径后重新运行命令。
   ```

2. **Gap Registry**
   路径：`docs/gap-registry.md`
   目的：了解现有条目结构和最大 Gap ID，确保新增条目格式一致、ID 不冲突。
   如果文件不存在，在变更摘要中注明"Gap Registry 尚不存在，将在确认后新建"。

3. **项目 CLAUDE.md**
   路径：`CLAUDE.md`（项目根目录）
   目的：了解现有规范内容，以便将新规范追加到合适章节，且不与现有规范重复。
   如果文件不存在，在变更摘要中注明"CLAUDE.md 尚不存在，将在确认后新建"。

### 1.2 提取预防措施条目

定位复盘文档的 **第 8 节：下次怎么做（Action Items / 预防措施）**，提取所有符合以下格式的条目（包括已勾选 `[x]` 和未勾选 `[ ]` 的）：

```
- [ ] [PREVENT] <预防措施描述>
      目标写入：Gap Registry / CLAUDE.md / 两者
      负责人：...
      截止日期：...
```

**识别规则**：

- 必须包含 `[PREVENT]` 标签才被提取为预防措施条目。
- 其他格式的 Action Item（不含 `[PREVENT]`）不执行写入，但在变更摘要中列出，提示人工关注。
- `目标写入` 字段是 Agent 执行写入的依据，人工可在确认阶段修改。
- 如果 `目标写入` 字段未填写或为"待填写"，在变更摘要中对该条目标记"目标写入未指定"，Agent 给出建议但不自行决定，由人工在确认阶段补充决策。
- 如果第 8 节整体为空或仍是占位符（"[需人工填写]"），停止执行并输出：
  ```
  复盘文档的第 8 节（Action Items）尚未填写，无内容可处理。
  请先完成复盘会议，填写预防措施后再运行此命令。
  ```

---

## Step 2：判断每条措施应写入哪里

对 Step 1 提取的每一条 `[PREVENT]` 条目，结合其 `目标写入` 字段和措施内容，判断写入目标。以下规则供参考：

| 写入目标 | 适用情形 | 写入位置 |
|---------|---------|---------|
| `Gap Registry` | 是对某类具体问题的预防性记录，需在未来 Sprint 审查中持续跟踪 | `docs/gap-registry.md`，末尾追加新条目 |
| `CLAUDE.md` | 是对 Agent 行为或团队工作规范的长期约束，应在后续会话中持续生效 | 项目根目录 `CLAUDE.md`，追加到适当章节 |
| `两者` | 既需要记录在案，又需要约束 Agent 行为，两处角度不同，内容可不完全一致 | 同时写入上述两处 |
| `workflow 文档` | 措施涉及某个 workflow 步骤的修改 | 对应 `workflow/*.md` 文件的相关步骤处 |

**无论判断结果如何，Agent 不自行执行写入，所有判断结果在 Step 3 的变更摘要中展示给人工审核。**

---

## Step 3：展示变更摘要，等待人工输入 "confirm"

**在执行任何写入操作之前**，完整输出以下格式的变更摘要。不允许在展示摘要之前修改任何文件。

```
========================================
update-from-retro 变更摘要
来源文件：<retro-file-path>
Sprint：<sprint-id>
提取到 [PREVENT] 条目：N 条
其他 Action Item（不处理，见末尾列表）：M 条
========================================

── 计划写入 docs/gap-registry.md ──────────────────────

[1] [PREVENT] <预防措施描述原文>
    负责人：...  |  截止日期：...
    写入内容预览：
    ┌─────────────────────────────────────────┐
    ### GAP-<ID>

    **类型**：预防性条目
    **来源**：<sprint-id> 复盘（<retro-file-path>）
    **描述**：<措施描述原文>
    **状态**：Active
    **创建日期**：YYYY-MM-DD
    **负责人**：<从复盘读取；未填写则标注 [需人工补充]>
    └─────────────────────────────────────────┘

[2] ...

（如无计划写入 Gap Registry 的条目，写"无"）

── 计划写入 CLAUDE.md ──────────────────────────────────

[3] [PREVENT] <预防措施描述原文>
    负责人：...  |  截止日期：...
    追加到章节：<章节名称>
    写入内容预览：
    ┌─────────────────────────────────────────┐
    - <规范描述原文>（来源：<sprint-id> 复盘，YYYY-MM-DD）
    └─────────────────────────────────────────┘

（如无计划写入 CLAUDE.md 的条目，写"无"）

── 目标写入未指定（需人工决策后再执行）─────────────────

[?] [PREVENT] <预防措施描述原文>
    Agent 建议：<Gap Registry / CLAUDE.md / 两者>
    建议理由：<一句话说明>
    请在 confirm 前告知应写入何处，否则此条将跳过。

（如无，写"无"）

── 其他 Action Items（不含 [PREVENT]，本命令不处理）───

- <条目原文>

（如无，写"无"）

── 注意事项 ─────────────────────────────────────────────

（列出任何疑似重复的情况，例如："[1] 疑似与 GAP-007 内容相近，请人工判断是否合并"）

========================================
以上变更将在收到 confirm 后执行。
如需修改某条目的写入目标，请先说明，再输入 confirm。
输入 confirm 执行 / 输入其他任何内容取消：
========================================
```

**等待用户输入。**

- 收到 `confirm` 才进入 Step 4 执行写入。
- 收到其他任何输入，输出"已取消，未写入任何文件"并终止。
- 如果变更摘要中存在"目标写入未指定"的条目，用户可在输入 `confirm` 之前先补充这些条目的写入目标。Agent 更新摘要后重新展示，再次等待 `confirm`。

---

## Step 4：执行写入，输出完成摘要

收到 `confirm` 后，按 Step 3 摘要中列出的内容依次执行写入。**只允许追加新内容，不允许修改或删除 Gap Registry 和 CLAUDE.md 中已有的任何内容。**

### 写入 docs/gap-registry.md

在文件末尾追加新的预防性 Gap 条目，格式与现有条目对齐。Gap ID 从现有最大 ID 自动递增（如 Gap Registry 为空则从 `GAP-001` 开始）。ID 若不确定，在摘要中标注 `[ID 待确认]`。

### 写入 CLAUDE.md

追加到现有的相关章节末尾。如果无合适章节，在文件末尾新增章节：

```markdown
## 规范更新（来源：Sprint <sprint-id> 复盘）

- <规范描述原文>（更新于 YYYY-MM-DD）
```

### 写入 workflow 文档（如适用）

在对应 workflow 文件的相关步骤处追加内容，并在行末标注来源：`（复盘更新，<sprint-id>，YYYY-MM-DD）`。

### 完成摘要

所有写入完成后，输出：

```
========================================
update-from-retro 执行完成
========================================

已写入：
  docs/gap-registry.md    ← 新增 N 条预防性条目（GAP-XXX 至 GAP-YYY）
  CLAUDE.md               ← 追加 N 条规范
  workflow/XX.md          ← 更新 N 处（如适用）

建议后续操作：
  1. 运行 git diff 检查写入内容是否符合预期
  2. 将变更提交到版本控制
  3. 在下次 Sprint 计划会议上确认各条措施的负责人和截止日期跟进情况
```

---

## 命令行为边界

- **不允许静默跳过条目**：所有 `[PREVENT]` 条目必须出现在变更摘要中。即使某条内容与已有条目疑似重复，也必须展示，并标注"疑似与 GAP-XXX 重复，请人工判断是否合并"，由人工决定。
- **不允许修改复盘文档本身**：`update-from-retro` 只写入 Gap Registry 和 CLAUDE.md，不修改作为输入的复盘文档。
- **不允许删除现有内容**：只追加新条目，不修改或删除 Gap Registry 和 CLAUDE.md 中已有的任何内容。
- **不允许修改现有 Gap 状态**：现有 Gap Registry 中已有条目的状态不在本命令职责范围内，不允许修改。
- **不允许无人确认自动执行**：任何情况下，未收到人工 `confirm` 都不执行写入。
