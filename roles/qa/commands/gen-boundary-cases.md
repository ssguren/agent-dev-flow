# /gen-boundary-cases — 生成边界情况测试用例集

## 使用说明

```
/gen-boundary-cases <endpoint>
```

示例：
```
/gen-boundary-cases POST /api/v1/auth/login
/gen-boundary-cases GET /api/v1/users/{id}
/gen-boundary-cases PUT /api/v1/orders/{orderId}/status
```

---

## Step 1：从接口规范中找到该接口的所有输入字段定义

从以下位置读取目标接口的字段定义（按优先级）：

1. OpenAPI / Swagger 文件（优先）
2. `docs/api-spec.md`
3. Controller 代码和对应的请求 DTO（作为补充）

提取每个输入字段的：
- 字段名
- 数据类型（String / Integer / Long / Decimal / Date / Boolean / Enum / List / Object）
- 是否必填
- 约束（最小长度 / 最大长度 / 最小值 / 最大值 / 格式 / 枚举值列表）
- 业务含义（用于判断哪些边界有业务价值）

如果字段定义缺失（类型未定义、约束未定义、枚举值未列出），在 Step 1 结束时列出问题清单，等待补充后再继续。

---

## Step 2：对每个字段系统性生成边界用例

以下类型必须覆盖，不得省略：

### String 类型

| 边界类型 | 测试输入值 | 说明 |
|---|---|---|
| null | `null` | 字段缺失或显式传 null |
| 空字符串 | `""` | 长度为 0 |
| 纯空白字符串 | `"  "` | 只含空格，不应等同于有效值 |
| 长度 = max | 恰好 N 个字符 | 边界合法值，应通过 |
| 长度 = max + 1 | N+1 个字符 | 超出限制，应被拒绝 |
| XSS 探针 | `<script>alert(1)</script>` | **安全相关** |
| XSS 探针变体 | `"><img src=x onerror=alert(1)>` | **安全相关** |
| SQL 注入探针 | `' OR 1=1 --` | **安全相关** |
| SQL 注入探针变体 | `"; DROP TABLE users; --` | **安全相关** |
| 全中文 | `你好世界` | 多字节字符，测试编码处理 |
| 全 emoji | `🎉🎊🔥💯` | 多字节 Unicode，测试存储和展示 |
| 混合特殊字符 | `!@#$%^&*()<>?/\|{}[]` | 特殊符号 |
| 路径穿越（文件名字段） | `../../etc/passwd` | **安全相关**，仅适用于文件名 / 路径类字段 |

### Number / Integer / Long 类型

| 边界类型 | 测试输入值 | 说明 |
|---|---|---|
| 零 | `0` | 是否允许零值（视业务而定） |
| 负数 | `-1` | 是否允许负数 |
| 最小值（规范定义） | min | 边界合法值，应通过 |
| 最小值 - 1 | min - 1 | 应被拒绝 |
| 最大值（规范定义） | max | 边界合法值，应通过 |
| 最大值 + 1 | max + 1 | 应被拒绝 |
| Integer 极大值 | `2147483647` | Integer.MAX_VALUE |
| 极大值 + 1 | `2147483648` | 溢出场景，Java 接受为 Long |
| 浮点数（若期望整数） | `1.5` | 类型不匹配 |
| 字符串形式 | `"123"` | 字符串传数字字段（框架兼容性） |

### Date / Time 类型

| 边界类型 | 测试输入值 | 说明 |
|---|---|---|
| null | `null` | 是否允许 null |
| 极早时间 | `1970-01-01T00:00:00Z` | Unix 纪元起点 |
| 极远未来 | `9999-12-31T23:59:59Z` | 极远未来时间 |
| 当前时间附近 | `now()` | 边界合法值 |
| 时区偏移 +08:00 | `2024-01-01T00:00:00+08:00` | 时区是否正确处理 |
| UTC 表示 | `2024-01-01T00:00:00Z` | 同一时刻的 UTC 表示 |
| 闰年 2 月 29 日 | `2024-02-29` | 合法闰年日期，应通过 |
| 非闰年 2 月 29 日 | `2023-02-29` | 应被拒绝 |
| 格式变体 | `20240101` / `2024/01/01` | 不符合规范格式，应被拒绝 |
| 夏令时切换点 | 视目标时区而定 | 跳过或重复的小时边界 |
| 过去时间（若需未来） | 昨天的日期 | 如截止日期不能是过去 |
| 未来时间（若需过去） | 明天的日期 | 如注册时间不能是未来 |

### List / Array 类型

| 边界类型 | 测试输入值 | 说明 |
|---|---|---|
| null | `null` | 整个数组为 null |
| 空数组 | `[]` | 长度为 0，视业务决定是否合法 |
| 单元素 | `[x]` | 最小非空数组 |
| 最大元素数（规范定义） | maxItems 个元素 | 边界合法值，应通过 |
| 超最大元素数 | maxItems + 1 个元素 | 应被拒绝 |
| 重复元素 | `[x, x, x]` | 是否去重或报错 |
| 包含 null 元素 | `[x, null, y]` | 应如何处理 null 元素 |
| 超大数组（性能） | 1000+ 元素 | 测试性能边界和超时 |

### Enum 类型

| 边界类型 | 测试输入值 | 说明 |
|---|---|---|
| 合法枚举值 | `"ACTIVE"` | 正常路径，应通过 |
| 非法字符串 | `"INVALID"` | 应被拒绝 |
| 小写变体 | `"active"` | 大小写敏感性测试 |
| 混合大小写 | `"Active"` | 大小写敏感性测试 |
| 空字符串 | `""` | 应被拒绝 |
| null | `null` | 是否允许 null |
| 数字形式 | `1` | 传数字代替字符串（框架兼容性） |

### Boolean 类型

| 边界类型 | 测试输入值 | 说明 |
|---|---|---|
| true | `true` | 标准布尔值 |
| false | `false` | 标准布尔值 |
| null | `null` | 是否允许 null |
| 数字兼容 - 1 | `1` | 部分框架将 1 视为 true |
| 数字兼容 - 0 | `0` | 部分框架将 0 视为 false |
| 字符串 true | `"true"` | 字符串形式（框架兼容性） |
| 字符串 false | `"false"` | 字符串形式（框架兼容性） |

## Step 3：输出表格格式

每条用例以表格输出，标注安全相关用例：

| 用例 ID | 字段 | 边界类型 | 测试输入值 | 预期 HTTP 状态 | 预期响应关键字 | 安全相关 | 优先级 |
|---|---|---|---|---|---|---|---|
| BC-001 | username | null | `null` | 400 | `field_required` | 否 | P1 |
| BC-002 | username | 空字符串 | `""` | 400 | `field_required` | 否 | P1 |
| BC-003 | username | 纯空白 | `"  "` | 400 | `field_required` | 否 | P1 |
| BC-004 | username | 长度 = max + 1 | 超过最大长度的字符串 | 400 | `max_length_exceeded` | 否 | P1 |
| BC-005 | username | XSS 探针 | `<script>alert(1)</script>` | 400 | `invalid_format` | **是** | P0 |
| BC-006 | username | SQL 注入 | `' OR 1=1 --` | 400 | `invalid_format` | **是** | P0 |
| BC-007 | age | 零值 | `0` | 400 | `out_of_range` | 否 | P2 |
| BC-008 | age | 负数 | `-1` | 400 | `out_of_range` | 否 | P1 |

---

## 安全相关用例汇总（单独列出，优先执行）

输出结束后，将所有"安全相关 = 是"的用例单独汇总，标注为优先执行：

```
## 安全相关用例（优先执行）

以下用例涉及安全边界，建议在回归测试中优先覆盖：

| 用例 ID | 字段 | 攻击类型 | 输入值 | 预期结果 |
|---|---|---|---|---|
| BC-005 | username | XSS | `<script>alert(1)</script>` | 400，输入被拒绝 |
| BC-006 | username | SQL 注入 | `' OR 1=1 --` | 400，输入被拒绝 |
```

## 规范缺失处理

如读取规范时发现以下情况，停止生成并输出问题清单，要求补充后再执行：

```
规范缺失，无法生成完整用例，需要补充以下信息：

1. 字段 `username` 未定义最大长度限制
2. 字段 `role` 的合法枚举值列表未列出
3. 400 错误响应的 body 格式未定义（无法确认"预期响应关键字"）
4. 接口未说明是否需要认证（无法判断 401 场景是否适用）

请补充规范后重新执行 /gen-boundary-cases
```
